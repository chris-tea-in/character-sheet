# Modifier Source Matrix — what can affect each block (2014 RAW)

The exhaustive map driving the Modifier Ledger rollout: for every stat block, each canonical 2014 source
that can affect it, the **effect channel** that carries it, and its **status**. Built by a per-block
fan-out (map + adversarial completeness critique) and cross-checked against the data.

**Status legend:** ✅ modeled (channel + representative data/compute exist) · 📝 data-gap (effect type
exists, data unauthored) · 🔧 system-gap (needs a new effect type/model) · ✋ manual (situational, player-
handled) · ⊘ out-of-scope. Companion to [DND_RULES_REFERENCE.md](DND_RULES_REFERENCE.md) and
[BACKLOG.md](BACKLOG.md) → Modifier Ledger.

## 🔧 System gaps — new effect types / models needed

1. **(Ability Scores (STR/DEX/CON/INT/WIS/CHA))** Per-effect cap flag on ItemEffect.ability_bonus / ability_set: RAW differs per item — Belt of Dwarvenkind caps at 20, Belt of Giant Strength/Amulet of Health do not. The model currently treats ability_bonus as uniformly uncapped (correct for some, wrong for Belt of Dwarvenkind).
2. **(Ability Scores (STR/DEX/CON/INT/WIS/CHA))** Permanent/consumable base-bump effect type: Manuals & Tomes permanently raise a score by +2 AND raise its maximum above 20, then the item is consumed. This is neither an active 'while worn' item effect nor a level-up ASI. Today only manual base editing represents it, with no ledger provenance.
3. **(Ability Scores (STR/DEX/CON/INT/WIS/CHA))** Effective-score 20 cap is enforced on BASE (level-up ASI) and on the FEAT step only, applied AFTER an uncapped racial step — so PC advancement can yield an effective score >20, and a feat's +1 can paradoxically lower an over-20 score. A single, ordered cap pass over the effective total is missing.
4. **(Ability Scores (STR/DEX/CON/INT/WIS/CHA))** No abilities entry in DerivedStats.breakdowns: the Modifier Ledger currently itemizes only speed/initiative/ac/proficiencyBonus. To roll the ledger out to abilities, deriveCharacterStats must emit a per-ability ModifierSource[] (base → racial → subrace → level-up-ASI → feat → item) the same way speed/initiative already do.
5. **(Ability Scores (STR/DEX/CON/INT/WIS/CHA))** Belt of Giant Strength has no per-variant catalog rows or effects — the ability_set channel exists but the six giant-type STR values (21/23/25/29) are unauthored data.
6. **(Armor Class)** Subclass/race unarmored-base-set effect: a structured 'set unarmored AC base' effect keyed to a source with an arbitrary ability modifier (Draconic Resilience 13+DEX, Lizardfolk 13+DEX, Tortle fixed 17, Loxodon 12+CON). Currently item-only via ItemEffect.unarmored_ac; Barbarian/Monk are hardcoded class-slug branches.
7. **(Armor Class)** Structured race-effect system (0/46 races have effects arrays): natural-armor AC bases live as prose in races.json base.traits with no application path.
8. **(Armor Class)** AC-floor / clamp model: Barkskin ('AC can't be less than 16') is a minimum clamp, not an additive bonus — no representation.
9. **(Armor Class)** Temporary/situational AC buff toggles: Mage Armor (sets base), Shield spell (+5 reaction), Haste (+2), Shield of Faith (+2) are all manual; a toggleable transient-AC layer would let these be tracked without write-time baking.
10. **(Armor Class)** Allocatable/dynamic item bonus: Defender weapon lets the player split a +3 across attack/damage/AC per turn — no model for reassignable bonuses.
11. **(Armor Class)** No-proficiency armor penalty: disadvantage on STR/DEX rolls + no spellcasting when wearing armor without proficiency is unmodeled (AC is still granted, which is correct).
12. **(Hit Points (max HP))** Retroactive-CON term in adjustedMaxHp: a render-time (effectiveCONmod − bakedCONmod)×totalLevel correction, OR rewrite stored maxHp whenever base CON / a CON-setting item changes — so ASIs, Amulet of Health, and CON half-feats taken after creation move max HP (BUG-57 / DND-HP1).
13. **(Hit Points (max HP))** Author the existing max_hp ItemEffect (amount + perLevel) on real catalog items — the channel is wired but 0/872 items use it.
14. **(Hit Points (max HP))** Structured race/subrace effect system (or at minimum generic hp_bonus_per_level wiring) so any subrace's per-level HP works, not just the hardcoded hill-dwarf slug (DND-RACE1).
15. **(Hit Points (max HP))** Per-class-level / per-subclass-level max-HP channel (Sorcerer Draconic Resilience +1/sorcerer-level) — multiclass-sensitive, no FeatureEffect variant today.
16. **(Hit Points (max HP))** Feat max-HP grants beyond Tough: a FEAT_EFFECTS entry or effects-array variant for Aberrant Dragonmark's per-level +1 (currently only its CON+1 asi is authored).
17. **(Hit Points (max HP))** Condition / max-HP-reduction model supporting MULTIPLICATIVE reduction (exhaustion 4 = max HP halved) and flat curse/level-drain reductions; adjustedMaxHp is additive-only today and there is no condition layer.
18. **(Speed)** FeatureEffect.speed — a class/subclass-feature speed channel (Barbarian Fast Movement, Monk Unarmored Movement, Ranger Roving, etc.), with per-level scaling tables and armor/shield gating (Fast Movement: no heavy armor; Unarmored Movement: no armor + no shield). FeatureEffect currently only carries ac/weapon_attack/weapon_damage.
19. **(Speed)** ItemEffect.speed_set — set/floor-to-value semantics for Boots of Striding & Springing ('walking speed becomes 30 regardless'), including precedence rules vs additive speed bonuses.
20. **(Speed)** speed_multiplier (item + spell) — multiplicative variant for Boots of Speed (x2), Haste (x2), Slow / exhaustion (x0.5), which the additive ledger cannot express.
21. **(Speed)** Heavy-armor STR-requirement penalty — derive-time computed -10 ft when wearing Chain Mail/Splint/Plate below the STR score, honoring the Dwarf exception. Data (strength_requirement) and STR are already available; only the computation is missing.
22. **(Speed)** Condition subsystem (exhaustion track + combat conditions) — exhaustion lvl2 halves / lvl5 zeroes speed; grappled/restrained/prone/stunned/etc. zero or restrict movement. No condition model exists.
23. **(Speed)** Alternate movement modes (climb/fly/swim speeds) and terrain-cost waivers (Boots of the Winterlands, Winged Boots, Slippers of Spider Climbing) — these are secondary speeds, not the single walking-speed number the block tracks today.
24. **(Initiative)** advantages.initiative dimension — AdvantageEntry and derived.advantages are {saves, skills} only (characterStats.ts:52,155); no way to record advantage/disadvantage on initiative for Feral Instinct, armor-without-proficiency, or any future feat/item.
25. **(Initiative)** Rollable initiative (RollKind 'initiative') — initiative is a static display value (CombatBlock.tsx:339-342); with no d20 roll, every advantage/die-adding source (Feral Instinct, Gift of Alacrity, Bless) is structurally unrepresentable as a roll modifier even if the data existed.
26. **(Initiative)** Half-PB-to-initiative class-feature term — Jack of All Trades (Bard 2) needs a derived bard-level-keyed PB/2 addend in effectiveInitiative; there is no FeatureEffect channel for initiative and no class-feature effect system that reaches this stat.
27. **(Initiative)** Structured class/subclass feature effect model for initiative — Feral Instinct and Jack of All Trades are prose features today; class-feature mechanics that touch initiative (numeric or advantage) have no authoring channel (FeatureEffect covers only ac/weapon_attack/weapon_damage).
28. **(Saving Throws (6))** FeatureEffect.save_proficiency — a STATIC class/subclass-feature channel granting save proficiency the class table doesn't (Monk Diamond Soul = all six saves; Rogue Slippery Mind = WIS; misc subclass save-prof features). FeatureEffect today carries only ac/weapon_attack/weapon_damage.
29. **(Saving Throws (6))** DERIVED save bonus channel for Paladin Aura of Protection — value = effective CHA modifier (recomputed every render, scales with CHA), not a flat amount. No existing flat-amount channel (ItemEffect.save / FeatEffect) can express a bonus whose value is another live ability modifier. Needs a 'bonus = f(other effective stat)' model.
30. **(Saving Throws (6))** Structured race/subrace effect system — racial save advantages are currently a hardcoded RACE_ADVANTAGES/SUBRACE_ADVANTAGES map in code (boolean, condition stripped), not derived from races.json (prose traits). 31. **(Saving Throws (6))** Condition-driven disadvantage on saves — getCharacterAdvantages has an advantage Set but NO disadvantage counterpart; exhaustion/restrained/paralyzed save penalties have no model (intentionally manual today, but a condition tracker would need a save-disadvantage channel).
32. **(Saving Throws (6))** Multiclass first-class save-proficiency enforcement — save dots are freely clickable with no class-lock; the RAW rule that save proficiencies come only from the first class is unenforced (soft gap; deliberate 'saves editable' design).
33. **(Skills (18) + passive Perception/Investigation)** half_proficiency effect/tier — a third proficiency level (or a separate computed layer) carrying Jack of All Trades floor(PB/2) on all non-proficient checks (skills + raw ability + initiative) and Remarkable Athlete ceil(PB/2) scoped to STR/DEX/CON, both suppressed when PB already applies. Round-down vs round-up differ.

37. **(Skills (18) + passive Perception/Investigation)** item skill-effect DATA authoring — the ItemEffect.skill channel exists and is wired, but 0/872 items carry it; flat-bonus items (Gloves of Thievery, +X Instrument) need their effects authored (data-gap, not system-gap).
